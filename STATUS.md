# סטטוס מערכת WebtopKids

**עודכן:** 03/03/2026

---

## ✅ מה עובד

| רכיב | סטטוס | הערות |
|------|--------|------|
| **משיכת נתונים (Scraper)** | תלוי התחברות | מריץ 2 תלמידות (יולי, אמי) כאשר `children_config.json` קיים |
| **דחיפה ל-VPS** | ✅ | `push_loop.mjs` דוחף כל 15 דקות + גיבוי על-demand |
| **שרת API** | ✅ | `/api/data`, `/api/status`, `/api/push`, וכו' |
| **לשוניות האפליקציה** | ✅ | 8 לשוניות: שיעורי בית, התראות, ציונים, יומן, אישורים, הודעות, קישורים, הכל |
| **מערכת התראות** | ✅ | Telegram + PTR (משוך לרענון) |
| **אישור שיעורי בית** | ✅ | כפתור "סמן כהושלם" → סטטוס + עדכון |
| **אישורים (חדש)** | ✅ | כפתור "אישרתי" → סטטוס מקומי "✓ אושר" |
| **הודעות כנקרא** | ✅ | בלחיצה על הודעה → מסומנת כ"✓ נקרא" |
| **רענון אוטומטי** | ✅ | כל 5 דקות (הופחת מ-15) |
| **GET ?refresh=1** | ✅ | מגדיר trigger ל-push_loop |
| **דדופליקציה** | ✅ | התראות כפולות מסוננות לפי type+student+subject+date |
| **מניעת דחיפת login page** | ✅ | אם `forgotPassword` בקישורים — לא דוחסים |

## ⚠️ דורש תחזוקה

| נושא | פעולה נדרשת |
|------|-------------|
| **פג תוקף התחברות** | הרצה: `WEBTOP_CAPTURE=true node webtop_scrape.mjs` — התחברות ידנית |
| **push_loop לא רץ** | הפעלה: `start_daemon.bat` או `fresh_pull.bat` במחשב הבית |
| **נתונים ישנים** | משוך לרענון באפליקציה או הפעל push_loop |

---

## 🔧 תיקונים שבוצעו

1. **אישורים** — כפתור "אישרתי" + API `/api/approval/done`
2. **הודעות** — סימון נקרא בלחיצה + API `/api/messages/read`
3. **רענון** — 5 דקות במקום 15, `refresh=1` מפעיל trigger
4. **דדופליקציה** — התראות כפולות בסקרייפר
5. **סינון login page** — לא דוחפים נתוני דף התחברות
6. **מערכת סטטוס** — `GET /api/status/system` מחזיר בריאות מערכת

---

## 📋 סדר הרצה מומלץ

1. **התחברות (פעם)**  
   `WEBTOP_CAPTURE=true node webtop_scrape.mjs`

2. **משיכה + דחיפה**  
   `fresh_pull.bat` — או `start_daemon.bat` ל-daemon קבוע

3. **דיפלוי לשרת**  
   `deploy_all.bat`

# WebtopKids — היכן הנתונים נשמרים

## קבצי JSON בשרת (בתיקיית הפרויקט)

| קובץ | תוכן |
|------|------|
| `homework_status.json` | סטטוס שיעורי בית (הושלם/לא), אישורים (אישרתי), הודעות נקראו |
| `data_cache.json` | נתונים שנתפסו מהאתר (התראות, ציונים, שיעורי בית, הודעות) |
| `children_config.json` | תמונות תלמידים (base64), כיתה, מקצועות תקפים |
| `special_events.json` | אירועים (ימי הולדת, ישיבות הורים) |
| `sent_reminders.json` | התראות שכבר נשלחו (למנוע כפילות) |
| `external_links.json` | קישורים חיצוניים (טפסים, דפי Webtop) |

## פרטים חשובים

### תמונות (Photos)
- נשמרות ב-`children_config.json` בתוך `children[].photo` כ-base64.
- נשמרות דרך `POST /api/children/:name/photo`.
- נטענות דרך `GET /api/children` (השרת מחזיר את ה-config כולל photo).

### שיעורי בית ישנים
- כל הנתונים נשמרים ב-`data_cache.json` (גם שיעורי בית ישנים).
- אין מחיקה אוטומטית של שיעורי בית ישנים — כל מה שהסקרייפר מושך נשמר.
- `homework_status.json` שומר אילו שיעורי בית סומנו כ"הושלמו".

### הודעות נקראו
- נשמר ב-`homework_status.json` תחת מפתחות `msg_...` עם `{ read: true }`.
- דרך `POST /api/message/read` עם `id` של ההודעה.

### סטטוס ישן
- הנתונים ב-`homework_status.json` לא נמחקים כשהסקרייפר מושך נתונים חדשים.
- עדכוני סטטוס (הושלם, אישרתי, נקרא) נשמרים ונשלפים בכל טעינת נתונים.

## דיאמון וסקרייפר

- `start_daemon.bat` — מפעיל את `push_loop.mjs` שרץ את הסקרייפר ומעביר נתונים לשרת.
- הנתונים החדשים נדחפים ל-`/api/push` ומעדכנים את `data_cache.json`.
- `homework_status.json` נטען ונשמר בנפרד — לא מתאפס כשמגיעים נתונים חדשים.
